Human-in-the-loop AI

Designing a triage system for collaboration between AI and humans

A Customer Support Triage System

Current Product

Customers submit support tickets via a support form and chatbot.
These tickets need to be triaged.
They must be prioritized, categorized and routed to the correct department.

Current Issues

Manual triage is slow and error-prone.
AI can help, but human oversight is essential due to edge cases, tones and complexity with sensitive issues.
Misclassification causes delays, frustrates users and could cause compliance issues.

The Opportunity

This project explores using a “Human in the loop” AI powered interface to address the current issues, where AI assists support agents with triaging tickets, to increase speed and accuracy.

Project Aims

Speed up triage with AI powered intelligent suggestions for priority, category and routing.
Make AI predictions transparent, and mistakes correctable in future.
Ensure high trust between support agent and the system.

1. Research and Inspiration

1.1 Competitor Analysis

Zendesk Triage

This tool automatically detects:

  1. Intent of the support ticket from a list
  2. Language used
  3. Sentiment of the customer

This information can be used to automatically:

  1. Route tickets to a specific department using rules
  2. Group similar requests for support agents
  3. Report on intent and sentiment over time

Freshdesk Smart Routing

This tool automatically detects:

  1. Type, priority and status of support request
  2. Allows customers to define custom fields

Human in the loop - agents can accept or reject the suggestions.
The system also learns and adapts over time from “corrections” made by support agents.

Front Helpdesk

This tool automatically detects:

  1. Adds type, priority and sentiment to support requests.
  2. Routes support requests to the right team member

Agents can adjust the tags added before responding or re-routing the request.

1.2 UX Pain Points

The solution should address the following common issues in AI-assisted UX:

  • Lack of Transparency: Why did AI choose this category, priority etc. ?
  • User Overfidence in Automation: Show agents a confidence rate to flag potential issues
  • Unlcear Override Process: Ensure its obvious how to fix AI's mistakes
  • No Clear Feedback Loop: The system should accept feedback and learn from human corrections

1.3 Key Patterns

I identified the following interaction patterns that will be important when creating our new support request triage system with the humans-in-the-loop design. They will help us to address the pain points identified.

Suggest & Confirm

AI clearly suggests tags, but humans can accept/reject.

Confidence Indicators

Model certainty on automated tags should be shown to agents.

Inline Feedback

Let the agent correct mistakes and give feedback.

Preview before Apply

Agents should clearly see the impact of any changes they make.

I then did an exploration of how these patterns could be incorporated into the new design to balance automation, with oversight and user control.

2.2 Clear Design Principles

Human First
AI should accelerate decisions, not make them for the user. The human always has final say.

Confidence Should Be Visible
Users deserve to know when the AI is not 100% sure. Confidence levels must be clearly communicated.

Correction is Collaboration
When a user corrects the AI, it means the system is learning and will work better next time. The interface should encourage feedback as a type of collaboration.

No Surprises
The user should at all times be aware of how their actions will affect the tickets they’re working on.

3. UX/UI Design

The final stage was to work on a medium fidelity prototype to be able to test the proposed system.

3.1 The Prototype

View prototype →

Explore how the agent reviews and confirms AI suggestions in the triage workflow prototype.

Throughout the prototype I used the four key interaction patterns identified in the research phase as part of the solution. I also transparently indicated where AI was responsible for the content shown in the dashboard.

Suggest & Confirm

  1. The AI tag suggestions are clearly marked as suggestions in the triage panel and in the dropdown box values.
  2. The AI summary is clearly flagged as being AI generated.
  3. Human-in-the-loop The customer service agent triaging the ticket has a choice to turn on and off each tag checkbox and confirm which of the tag suggestions they accept.

Confidence Indicators

  1. The confidence level in the tag suggestions is clearly shown at the top of the triage panel in the sidebar.
  2. The confidence level is also shown beside the values when the dropdown boxes are opened.
  3. An emoji is also shown as a shorthand easily identifiable icon to explain the confidence level.

Inline Feedback

  1. The user is asked to “Accept” the suggestions, showing they can give feedback they are correct or not.
  2. The user is then thanked for their feedback, and told the feedback helps the system learn encouraging future feedback.

Preview before Apply

  1. When the user checks on or off a suggested tag value in the AI Suggestions box the associated dropdown field updates its value and has a highlight effect to draw attention and show the user the effect of their actions.
  2. The user can preview the result of their actions before applying the changes using the “Save” button beneath the dropdown boxes.

4. Reflections and Next Steps

Key Learnings

  1. for AI is about designing a collaborative product that involves human process and automation together.
  2. It is crucial to make AI’s suggestions transparent and correctable.
  3. Users should be able to trust the system they are using, but also feel in control of the final outcome.
  4. Human-in-the-loop design should allow various levels of AI involvement across different parts of a product which will evolve over time as technology improves.

What I’d do next?

  1. I’d add in extra information so users could see the reason AI came to the conclusions it did on both the Tag Suggestions and the Ticket Summary text. Showing users the reason behind it will build trust and transparency.
  2. I’d design a triage flow that allows multiple similar tickets to be triaged at once.
  3. I’d add in AI assistance in the “Reply” area of the dashboard where agents reply to the support request.
  4. A mobile-friendly triage view that will allow for triage of urgent requests on the go.
